# Анализ данных в онлайн торговле
## Описание проекта
В репозитории представлен проект по комплексному анализу транзакционных данных интернет-магазина на основе датасета [Online Retail Dataset UCI](https://www.kaggle.com/datasets/carrie1/ecommerce-data).

Проект нацелен на реализацию стандартных методов в сфере e-commerce: от очистки и EDA до когортного анализа и симуляции A/B-теста.

## Цели и задачи
### Цели проекта:
* провести разведочный анализ данных (EDA);
* дать оценку продукту исходя из статистического анализа показателей;
* отработать сценарий проведения A/B-теста.
### Задачи:
* очистка и предобработка данных;
* исследование пользовательской активности;
* анализ распределения показателей и их визуализация;
* расчет важнейших продуктовых метрик;
* сегментация клиентов с помощью RFM-анализа;
* проведение когортного анализа удержания клиентов;
* симуляция A/B-тестирования.

## Описание данных
Для проекта использовался датасет Online Retail Dataset UCI. Данные содержат транзакции интернет-магазина за период с декабря 2010 по декабрь 2011 г.
### Исходные признаки:
|    Поле     |          Описание          |
|-------------|----------------------------|
| InvoiceNo   | Номер заказа               |
| StockCode   | Код товара                 |
| Description | Описание товара            |
| Quantity    | Количество товара          |
| InvoiceDate | Дата и время транзакции    |
| UnitPrice   | Цена товара                |
| CustomerID  | Идентификатор пользователя |
| Country     | Страна пользователя        |
### Особенности данных:
* присутствуют возвраты товаров;
* имеются пропущенные значения;
* распределения некоторых величин имеют длинный правый хвост;
* наблюдаются экстремальные значения.

## Предобработка
Предварительная обработка данных включала:
### Очистка данных
* строки с отсутствующим `CustomerID`;
* дубликаты;
* некорректные значения цены и количества.
### Feature engineering
Были созданы дополнительные признаки:
* выручка `Revenue = Quantity * UnitPrice`;
* месяц заказа `YearMonth`.

## Результаты
По итогам анализа получены следующие результаты:
### EDA
* выявлены наиболее прибыльные страны и товары, определена динамика выручки во времени;
* проведен анализ клиентской базы и сегментация по LTV, подсчитаны DAU, WAU, MAU;
* выявлено сильное смещение распределений переменных.
### Продуктовые метрики
* Average Order Value
* Average Revenue per User
* Purchase Frequency
* Average Time between Purchases
* Repeat Purchase Rate
* Churn Rate
### RFM-анализ
* клиенты разделены на сегменты по давности, частоте и сумме покупок;
* определены стратегии дальнейшего взаимодействия с каждым сегментом.
### Когортный анализ
* сформированы когорты клиентов по признаку месяца привлечения;
* выявлено быстрое снижение Retention rate после первого месяца.
### A/B-тестирование
* проведена симуляция A/A-теста на корректность "системы тестирования";
* проведена симуляция A/B-теста с добавлением эффекта;
* выявлено отсутствие статистически значимого изменения целевой метрики при добавлении эффекта.

## Ограничения
* датасет не содержит данных о просмотрах, кликах, корзинах и т. п.;
* часть пользователей отсутствует из-за пропусков `CustomerID`;
* A/B-тест является симуляцией, а не реальным экспериментом;
* данные относятся к ограниченному периоду времени, и их не хватает для полноценного проведения эксперимента;
* отсутствует информация о маркетинговых каналах и источниках трафика.
